3.4. Goals and guidelines

Following is a list of the most important goals and guidelines than must be ensured during implementation.

The KISS principle (keep it simple stupid!) - architecture, algorithms, mechanism used during implementation should be as simple as possible. This way product easy to maintenance and extension will be created.

Don't early optimize - optimalization can be done only in case that it's really needed (it must be proved in tests). Implementation should focus on correctness of working and easines of reading the code. Speed is not important.

Look and feel are very important - there should be a possibility to modify theme. Operations done with user interaction should work as fast as possible. User should not have to wait for results. Time from event to first results should be minimized.
