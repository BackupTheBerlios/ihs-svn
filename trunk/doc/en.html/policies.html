4.7. Policies and guidelines

4.7.1. Used software

Following software is used to develop system:

  • java >= 1.4.2

  • ant >= 1.6

  • Oracle >= 10g

  • PostgreSQL >= 8

  • Eclipse >= 3.0

4.7.2. Filesystem structure

./doc. Directory including all documentation and files used to generate it.

./doc/en.src. DocBook source files which are used to generate English documentation.

./doc/en.rtf. Documentation in RTF format. Generated automatically.

./doc/en.html. Documentation in HTML. Generated automatically.

./doc/imgs. Common graphic files.

./doc/pl.src. DocBook source files which are used to generate Polish documentation.

./doc/pl.rtf. Documentation in RTF format. Generated automatically.

./doc/pl.html. Documentation in HTML format. Generated automatically.

./src/com/foo_baz/ihs. Source code of main part of the system. Code responsible for communication with Virtuam Qmail, database handling, etc.

./src/com/foo_baz/ihs/mailservice. Wrappers for some types from Virtual Qmail.

./src/com/foo_baz/ihs/backing. Backing beans for main part.

./src/com/foo_baz/ihs/backing/mailservice. Backing beans for mail service, at this time for Virtual Qmail.

./src/com/foo_baz/v_q. Code generated from IDL files distributed with Virtual Qmail.

./src/com/foo_baz/util. Utilities. Common code.

./src/com/corejsf. Code taken from GEA-2004.

./build. Directory where compiled files are stored.

./pkg. Includes files used to generate distribution packages.

./pkg/rpm. Files for building RPM packages.

./tmp. Temporary files. This directory is not versioned.

./WebContent. JSF pages, CSS stylesheets, graphic files. Files server by J2EE.

./WebContent/mailservice. User interface for managing mail service.

./WebContent/META-INF. Configuration files for J2EE server.

./WebContent/WEB-INF. Application's configuration.

./WebContent/WEB-INF/lib. Libraries needed for compilation.

4.7.3. Compilation

Building this system is very easy, all you need to do is execute following command in project's root directory: 

ant war

it creates a WAR file in ./build. 

ant deploy

it tries to install application using configuration written in build.properties. 

ant deploy-to-dir

it copies WAR file to Tomcat's directory. 

ant doc

when inside ./doc directory it generates documentation.

4.7.4. Creating packges

Command: 

ant pkg-rpm

generates RPM package. 

ant pkg-tgz

generates compressed tarball with code.

When you have TAR/GZ you can easily create RPM package, just run: 

rpmbuild -tb ihs-*.tar.gz
