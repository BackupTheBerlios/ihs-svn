Chapter 2. Introduction to the system

IHS will be a platform for e-mail and hosting systems for different environments - small and medium companies for their employees or service providers offering free or paid services.

It's a complicated task to build such system - it requires integration of many existing applications, i.e. - following systems must be glued together - WWW server, MTA, FTP server, database. It must be also possible to integrate system with some other applications that are specific to client's environment.

Framework must be build that will allow to administrate all those applications throught one, unified interface. It will be a centralized environment for users, administrators and developers. Offering common functions independently of elements used, also API should be generic and flexible enough to allow sophisticated software to be build on a base of created system.

IHS will be based on following applications and technologies:

  • Apache as a WWW server;

  • PostgreSQL as a default database;

  • ProFTPD as a default FTP server;

  • qmail and Virtual Qmail as MTA;

  • CORBA as a services' communication protocol;

  • J2EE for the integrated management system.

Many other applications will be used to build the system. List is very long, only the most important were listed above.

Implementation is based only on an open source or free software, all created artefacts will be released also as the source code. This will guarantee maximum of flexibility for users. foo-baz.com company offers services related with installation, integration or maitenance of IHS in client's environment.
