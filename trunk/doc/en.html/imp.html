4.8. Implementation details for IHS

4.8.1. Core of the system

Figure 4-6. Package com.foo_baz.ihs

Main part of the system resides in com.foo_baz.ihs package, picture Figure 4-6 shows elements included in this package. Followin is a short description of each of them:

Configuration. This is an interface to configuration - it allows to add, update or remove parameters. By default configuration is kept in database. Interface handles only strings.

Service. This interface must be implemented by any plug-ins for IHS. It's used for communication between plug-in and system.

MailService. Interface representing plug-in handling mail system. It includes some additional methods.

Administrators. Interface representing service managing administrators's accounts.

IncredibleSystemHosting. Core of the system. Implements Administrators interface. Includes code managing plug-ins, implements Configuration.

Administrator. Represents information stored about administrator.

ExtendedAdministrator. Administrator extended with some data used by GUI.

4.8.2. User interface

Figure 4-7. Package com.foo_baz.ihs.backing

Figure 4-8. Package com.foo_baz.ihs.backing.mailservice

Figure 4-7 and Figure 4-8 include elements that are used as a mediator between core of the system and user interface represented by JSF pages. Elements have different scope, some are created for every request, some for every session. Usually any JSF pages uses one of them.

4.8.3. Plug-ins

Here's list of plug-ins installed with IHS:

4.8.3.1. VirtualQmailMailService

Implements MailService interface. Uses Virtual Qmail as a backend for mail system. Communication is CORBA based.

Files in packages com.foo_baz.ihs.mailservice and com.foo_baz.ihs.VirtualQmailMailService include implementation of this plug-in.
