3.1. Installation

3.1.1. Downloading

It's worth checking project's page.

Released files are kept on SourceForge. Check if there's newer version.

You can also access repository with newest version - http://svn.berlios.de/svnroot/repos/ihs/trunk

3.1.2. Dependencies

Main dependencies:

  • Java Runtime at least 1.4;

  • J2EE application server, at this time system is tested only with Tomcat 5;

  • Virtual Qmail at least 11;

  • Oracle or PostgreSQL database (can be different than instance used by Virtual Qmail);

3.1.3. Compilation, packaging

Compilation is described in Section 4.7.3.

3.1.4. Deploying on J2EE server

3.1.4.1. Tomcat

The easiest way is to copy WAR to webapps directory in Tomcat installation

3.1.4.2. Other application servers

There's no support for other J2EE servers. In future versions there will be support for other popular applications.

3.1.5. Database setup

After IHS is installed you should setup database link and create object in database that are needed by IHS.

IHS uses jdbc/ihsDB resource as a link to database. Resource is looked up in JNDI directory.

JNDI configuration differs betweem servers. Example files for Tomcat are located in META-INF directory.

You should remember that JDBC drivers are not distributed with IHS. You must aquire them from other sources. For Tomcat you can usually put them into common/lib directory.

Use oracle.sql or pgsql.sql scripts to create objects in database.
